    常见的应用层协议

    HTTP：超文本传输协议，用于客户端浏览器和服务端进行通信

    HTTPS：超文本传输安全协议，在HTTP的基础上通过传输加密和身份认证保障安全性

    TELNET：远程登录协议

    DNS：域名解析协议

    SSH：为远程登录会话和其他网络服务提供安全性的协议

    FTP：文件传输协议

    SMTP：面向连接的简单邮件传输协议，提供写信，传送，报告传送状态，显示信件等功能

    HTTPS如何提供安全性传输:

    客户端浏览器向服务器发起SSL安全连接请求；

    服务器发送带有证书的响应；

    客户端向信任的证书签发中心认证该证书，通过认证则进行下一步，否则由用户来确认是否进行连接；

    客户端认证服务端身份后，服务端要求客户端发送证书，服务端检查无误并取得证书中的公钥；

    自此，服务端客户端双方的身份验证结束。

    JAVA和C++的区别

    1. Java是面向对象的语言，所有对象均继承自java.lang.object，而C++为了兼容C既支持面向对象也支持面向过程
    2. Java通过虚拟机实现跨平台特性，而C++依赖特定的平台
    3. Java没有指针，引用可以理解为安全指针， 而C++继承了C的指针
    4. Java支持自动垃圾回收， 而C++需要手动回收
    5. Java不支持多重继承，通过实现多个接口来达到相同目的，而 C++支持多重继承
    6. Java不支持操作符重载， 而C++支持
    7. Java的保留字goto不可用，而 C++的goto可用

    JAVA大的三大特性：封装，继承和多态

    封装是指隐藏对象的具体属性和细节实现，仅对外提供公共访问方法；

    封装的目的是提高安全性和简化编程，使用者不需要知道调用的具体实现。

    继承是指将相同的属性和方法提取出来构建一个父类，具有该属性和方法的子类可以继承该父类，子类只能继承一个父类，子类可以重写overwrite从父类继承的方法；

    继承的目的是提高代码复用。

    多态可以分为设计时多态和运行时多态，设计时多态是指在一个类中可以定义多个同名但调用参数不同的方法，根据传入的参数类型和顺序决定调用，运行时多态是指子类可以重写从父类继承的方法，根据运行时对象的类型决定调用哪个方法；

    多态的目的是增加代码的灵活度。

    final关键字

    • 修饰基本数据类型变量：必须初始化且不能再改变，常用static一起使用成为常量
    • 修饰引用变量：不能再指向其他对象
    • 修饰方法：方法不可被重写
    • 修饰类：类不能被继承

    static关键字

    • static修饰的变量称为静态变量，只在类加载时初始化一次，不会因为对象的创建而重新加载。

    transient关键字

    实现了Serilizable接口的类创建的对象，可以被序列化。而对象中被transient关键字修饰的成员变量不会被序列化保存。

    抽象类 vs 接口类

    抽象类和抽象方法使用abstract关键字进行声明。抽象类至少有一个抽象方法，抽象类不能被实例化，只能被继承。

    在Java8之前，接口类可以看成是完全抽象的抽象类，从Java8开始，接口类也可以有默认的方法实现：这是因为维护完全抽象的接口成本太高，想要在接口中新增一个方法，就要在所有实现该接口的类中实现新增的方法。

    区别：

    • 从设计层面上看，抽象类是IS-A的关系，所有子类对象必须能够替换父类对象；而接口类是LIKE-A的关系，只是提供一种方法实现契约。
    • 从使用上看，一个类可以实现多个接口，但只能继承一个抽象类
    • 接口类的所有字段都必须是static final的，而抽象类没有这种限制
    • 接口类的所有成员都必须是public的，而抽象类没有这种限制

    JAVA容器

    容器类型

    Java容器主要包括Collection和Map，Collection用于对象的存储，Map用于键值对的存储

    Collection

    • Set

      1. TreeSet：基于红黑树实现，支持有序性操作，例如根据一个范围查找元素，但查找效率不如HashSet；O(logn)
      2. HashSet：基于哈希表实现，支持快速查找但不支持有序性操作，丢失了元素的插入顺序信息；O(1)
      3. LinkedHashSet：具有HashSet的查找效率，内部使用双向链表维护插入顺序

    • List

      1. ArrayList：基于动态数组实现，支持随机访问
      2. Vector：和ArrayList相似，但它是线程安全的
      3. LinkedList：基于双向链表实现，只能顺序访问，但是可以快速地向链表中插入和删除元素，可用于实现栈，队列，双向队列

    • Queue

      1. LinkedList：可用于实现双向队列
      2. PriorityQueue：基于堆结构实现，可用于实现优先队列

    Map

    • TreeMap：基于红黑树实现
    • HashMap：基于哈希表实现
    • ConcurrentHashMap：和HashMap相似，但线程安全，引入了分段锁效率更高
    • LinkedHashMap：使用双向链表来维护元素的顺序，顺序为插入顺序或者最近最少使用(LRU)顺序

    设计模式

    迭代器模式(Iterator)

    ​ Collection继承了Iterable接口，所有实现Collection接口的类都能使用Iterator()方法创建的Iterator对象，迭代遍历集合中的元素

    适配器模式(Adapter)：把一个类的接口变成另一种接口，从而使原本因接口不匹配而无法一起工作的两个类能够一起工作。

    java.util.Arrays的asList()方法能将Array转换成List

    java

    ArrayList vs LinkedList

    ArrayList是基于动态数组实现的，LinkedList是基于双向链表实现的。它们的区别可以归结于数组和链表的区别：

    • ArrayList：数组支持随机访问，插入和删除的代价很高，需要移动大量元素

    • LinkedList：链表只能顺序访问，插入和删除只需要改变指针

    HashMap vs HashTable vs ConcurrentHashMap

    • HashMap底层是数组+链表实现，不是线程安全的；初始size为16，每次扩容newsize=oldsize*2；key和value可以为null
    • HashTabel底层是数组+链表实现，是线程安全的，实现线程安全的方法是修改数据会锁住整个HashTable，使得其他线程都不能访问直到锁被释放；初始size为11，每次扩容newsize=oldsize*2+1；key和value都不能为null
    • ConcurrentHashMap底层是分段的数组+链表实现，也是线程安全的，通过分段锁的机制提供效率更高的线程安全。将整个Map分为N个Segement，线程访问修改可以对单个Segment加锁，因此效率N倍

    带有颜色属性的二叉查找树，也称为了自平衡二叉查找树

    每个节点都被标记为红色或者黑色

    进程与线程

    进程

    进程是资源分配的基本单元。PCB(进程控制块)保存着进程的基本信息和运行状态，进程的创建和撤销实际上是对PCB的操作。

    线程

    线程是独立调度的基本单元。一个进程可以有多个线程，它们共享进程的资源。

    例如QQ和浏览器是两个不同的进程，浏览器可以有多个线程用于发起Http请求，事件响应，渲染页面等任务。线程的并发执行允许用户在浏览器点击一个新链接发起http请求的同时，还能响应用户的其他操作。

    进程 vs 线程

    • 资源：进程是资源分配的基本单元，线程不拥有资源但是可以访问隶属进程的资源。
    • 调度：线程是独立调度的基本单元，在同一进程内，切换线程不会引起进程切换，而从一个进程中的线程切换到另一个进程中的线程，会引起进程切换。
    • 系统开销：创建和撤销进程需要系统重新分配和回收资源，开销远大于创建和撤销线程；并且进程间切换需要保存和重新配置环境，而线程切换只需要修改少量寄存器内容，开销很小。
    • 通信：线程通过修改共享的进程资源来完成通信，而进程通信需要借助IPC。

    进程状态的切换

    Java线程

    有三种使用线程的方法：

    1. 实现Runnable接口
    2. 实现Callable接口
    3. 继承Thread类

    实现接口 vs 继承类

    实现接口会更好一些，因为

    • Java不支持多重继承，继承了Thread类就不能在继承其他类，但可以实现多个接口
    • 继承整个Thread类开销太大

    锁机制

    Java提供两种锁机制，用于控制多个线程对共享资源的互斥访问，分别是JVM实现的Synchronized和JDK实现的ReentrantLock。

    java

    Synchronized vs ReentrantLock

    1. 实现机制不同，Synchronized是JVM实现的而ReentrantLock是JDK实现的
    2. 当持有锁的线程长期不释放锁时，等待线程可以选择放弃等待，先去处理其他任务。ReentrantLock可中断而Synchronized不可
    3. 公平锁是指等待同一个锁的所有线程，按照申请锁的时间顺序依次获得锁，Synchronized中的锁是非公平的，ReentrantLock的锁默认是非公平的，也可以公平
    4. 一个ReentrantLock锁可以同时绑定多个condition对象

    在不需要使用到ReentrantLock的高级功能时，默认使用Synchronized来实现锁。因为Synchronized是JVM实现的锁机制，JVM原生支持，而ReentrantLock不是所有版本的JDK都支持；另外Synchronized不用担心没有释放锁导致死锁问题，JVM会确保锁的释放。

    乐观锁 vs 悲观锁

    乐观锁假设一般情况下不会发生冲突，所以在数据提交更新时，才会进行冲突检测。如果发现冲突了，则返回用户错误信息，由用户决定如何处理；

    悲观锁会假设最坏的情况，在修改数据前先申请获得锁。

    GC

    对象是否可回收

    判断对象是否可回收，主要有两种方式：

    1. 引入计数法：每个对象创建时都引入一个计数器，对象被引用时计数器加1，引用失效时计数器减1，当计数器为0时对象就是不再被使用到的，可以被JVM回收。但是主流垃圾收集器没有采用这种方式，因为这种方式很难解决循环依赖问题。
    2. 可达性分析：将每个对象看作是节点，并且定义了一些对象作为根节点GC root，如果一个对象引用了另外一个对象，那么代表第一个对象的节点就会有一条指向第二个对象的边。JVM会起一个线程从所有的GCroot开始遍历，遍历完所有节点后发现有些节点不可达，那么就认为这些节点没有用了，可以被回收。

    垃圾回收算法

    JVM将堆内存分为新生代，老生代和永生代，根据对象的生命周期将其存储到不同区域，使用不同的垃圾回收算法：

    1. 复制算法：新生代使用复制法；复制法是指将内存空间分为两块，每次创建对象都会将其存储在其中一半，当其中一半内存用完时，就会出发minorGC将还存活的对象依次复制到另外一半，然后对已使用的内存一次回收清理。每次只使用一半的空间，内存使用率低。新生代实际使用复制算法时是按照8:1:1的比例分为三块，分别为Eden区，S0区，S0区，新创建的对象放在Eden区，Eden内存不够时触发minorGC，将存活的对象复制到S0区。S区不够用时会有新生代对象进入老生代
    2. 标记-清除算法和标记-整理算法：老生代使用这两个算法。标记所有需要回收的对象，在标记完成之后统一回收标记的对象。这个算法的有两点不足：1. 效率问题，标记和清除的效率都不高；2. 空间问题，对象清除之后将导致内存中产生大量不连续的内存碎片，导致创建大对象时无法分配连续的内存空间。整理法的区别在于标记完成之后不是清理对象，而是将存活的对象向一端移动，该方法能避免产生内存碎片。
    3. 分代收集算法：根据对象的生命周期将内存分为新生代和老生代，新生代使用复制法，老生代使用标记-清除法或标记-整理法。

    三种GC

    • minorGC：Eden区空间不足时触发，清理Eden区
    • majorGC：老年代空间不足时触发，清理老年代
    • fullGC：调用System.gc时，清理整个堆空间-包括新生代和老年代
